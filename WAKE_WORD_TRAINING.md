# Custom Wake Word Training

This document describes how to train a custom openWakeWord model — specifically how the "Computer" wake word in this repo was built, and how to train your own.

The trained model (`models/computer.onnx`) is already included and ready to use. You only need this guide if you want to train a different wake word or retrain with more voice samples.

## Overview

Training uses [openWakeWord](https://github.com/dscripka/openWakeWord) with synthetic audio generated by [piper-sample-generator](https://github.com/dscripka/piper-sample-generator) (a fork of Piper TTS). The process is:

1. Generate thousands of synthetic TTS samples of your wake word phrase
2. Augment with room impulse responses and background noise
3. Train a small DNN classifier on top of pre-extracted audio features
4. Export to ONNX

Typical training time: ~2-3 hours on an RTX 3090 for 50,000 steps.

## Requirements

Training uses a **separate Python environment from the runtime** — Python 3.10 is required (openWakeWord's training dependencies are incompatible with 3.9).

System packages:
```bash
sudo apt install python3.10 python3.10-venv ffmpeg
```

GPU: An NVIDIA GPU is strongly recommended. Training on CPU is possible but very slow.

## Step 1: Set up the training environment

```bash
# Create a Python 3.10 venv (separate from the runtime stt-env)
python3.10 -m venv ~/oww-env
source ~/oww-env/bin/activate

# Clone openWakeWord
git clone https://github.com/dscripka/openWakeWord ~/openWakeWord
cd ~/openWakeWord
pip install -e ".[train]"

# Clone piper-sample-generator (for synthetic TTS)
git clone https://github.com/dscripka/piper-sample-generator ~/piper-sample-generator
cd ~/piper-sample-generator
pip install -r requirements.txt
```

Note: `piper-sample-generator` downloads Piper TTS voice models on first use (~500MB). They are cached in `piper-sample-generator/models/`.

## Step 2: Download required data files

openWakeWord training requires pre-computed audio features and validation data. Download from the [openWakeWord releases page](https://github.com/dscripka/openWakeWord/releases):

```bash
mkdir -p ~/oww-training
cd ~/oww-training

# Pre-computed negative features (~4GB) - audio features from ACAV100M dataset
# Find the current download URL on the openWakeWord releases page
wget -O openwakeword_features_ACAV100M_2000_hrs_16bit.npy \
    <URL from releases page>

# Validation set features
wget -O validation_set_features.npy \
    <URL from releases page>
```

You also need audio for augmentation:

**Room impulse responses** (reverb simulation):
```bash
mkdir -p ~/oww-training/mit_rirs
# Download MIT IR Survey dataset or another RIR dataset
# The openWakeWord repo has links in its training documentation
```

**Background noise** (for mixing):
```bash
mkdir -p ~/oww-training/fma
# Download Free Music Archive (FMA) small dataset or similar background audio
# Any collection of diverse audio files works
```

## Step 3: Record real voice samples (recommended)

Synthetic TTS samples alone work, but adding real recordings of your own voice significantly improves accuracy and reduces false positives for your specific voice.

Record 20-50 clips of yourself saying the wake word naturally:
- Vary your tone, speed, and distance from the mic
- Record in different environments (quiet room, with background noise, etc.)
- Each clip should be 0.5-2 seconds, 16kHz mono WAV

```bash
# Example: record 5 seconds, trim to the word
arecord -f S16_LE -r 16000 -c 1 sample_001.wav
```

The 30 real recordings used to train the included "Computer" model are in `training/real-samples/`. You can use them as-is, supplement with your own voice, or replace them entirely.

Place your recordings in a directory you'll reference from the training config.

## Step 4: Configure training

Edit `training/computer_model.yaml` (or copy and rename it for a new wake word):

```yaml
model_name: "my_wake_word"

target_phrase:
  - "my wake word"   # The phrase to detect

custom_negative_phrases:
  - "similar phrase"  # Phonetically similar phrases that should NOT trigger
  - "another phrase"

# Update all paths to match your system:
piper_sample_generator_path: "/home/you/piper-sample-generator"
output_dir: "/home/you/oww-training/my_wake_word_output"
rir_paths:
  - "/home/you/oww-training/mit_rirs"
background_paths:
  - "/home/you/oww-training/fma"
feature_data_files:
  "ACAV100M_sample": "/home/you/oww-training/openwakeword_features_ACAV100M_2000_hrs_16bit.npy"
false_positive_validation_data_path: "/home/you/oww-training/validation_set_features.npy"
```

Key parameters:
- `n_samples`: Number of synthetic TTS samples to generate. 5000 is a good starting point.
- `steps`: Training steps. 50000 worked well for "Computer". More steps = longer training but diminishing returns.
- `target_false_positives_per_hour`: Controls the sensitivity/false-positive tradeoff during training.
- `custom_negative_phrases`: Important — add words that sound similar to your wake word so the model learns to distinguish them.

## Step 5: Run training

```bash
source ~/oww-env/bin/activate
cd ~/openWakeWord

python openwakeword/train.py --training_config /path/to/your_model.yaml
```

If you have real voice samples to include, add them to the training data after synthetic generation. The openWakeWord training script accepts a `--custom_positive_wav_dir` argument pointing to a directory of real WAV files:

```bash
python openwakeword/train.py \
    --training_config /path/to/your_model.yaml \
    --custom_positive_wav_dir /path/to/real-samples/
```

Training output goes to the `output_dir` specified in the config. You'll find:
- `<model_name>.onnx` — the model file for use with dictation.py
- `<model_name>.tflite` — TFLite version (not used by this project)
- Various `.npy` feature files used during training

## Step 6: Deploy the model

Copy the trained `.onnx` file to the dictation repo's `models/` directory:

```bash
cp ~/oww-training/output/your_model.onnx ~/local-dictation/models/your_model.onnx
```

Then update `dictation.py` to point to your model. In the `WAKE_WORD_MODEL_PATHS` list, add your model path, and update the `wake_words` parameter in the config dict to match:

```python
WAKE_WORD_MODEL_PATHS = [
    os.path.join(SCRIPT_DIR, "models", "your_model.onnx"),
    ...
]

# In the config dict:
wake_words="your wake word",
```

Also copy the model into openwakeword's package directory so it's found at runtime:

```bash
cp models/your_model.onnx \
    ~/stt-env/lib/python3.9/site-packages/openwakeword/resources/models/
```

Restart dictation.py and test.

## Tuning sensitivity

If the wake word triggers too easily (false positives), increase `wake_words_sensitivity` in `dictation.py` (range 0-1, higher = less sensitive).

If it misses real activations, decrease it.

The model included in this repo uses `wake_words_sensitivity=0.7`, which was tuned to balance responsiveness against false triggers in a typical home office environment.

## Files in this repo

```
training/
  computer_model.yaml     Training config (update paths before use)
  real-samples/           30 real voice recordings of "Computer" used for training
    computer_001.wav
    computer_002.wav
    ...
    computer_030.wav
models/
  computer.onnx           Final trained model (ready to use, no training required)
```

The `real-samples/` recordings are single-word utterances at 16kHz mono, recorded on a Scarlett 2i2 USB interface. They represent one person's voice across varied tone and pacing. When training for your own voice, supplementing or replacing these with your own recordings will yield better results.
